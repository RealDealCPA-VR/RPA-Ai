# LLM Integration Guide

## Overview

This AI-Powered RPA system is specifically designed to work seamlessly with Large Language Models (LLMs). This guide explains how to integrate the system with LLMs for intelligent automation.

## Why LLM Integration?

1. **Natural Language Control**: Users can describe automation tasks in plain English
2. **Intelligent Workflow Generation**: LLMs can create complex workflows from simple descriptions
3. **Adaptive Automation**: LLMs can modify workflows based on execution results
4. **Error Recovery**: LLMs can analyze failures and suggest fixes
5. **Workflow Optimization**: LLMs can improve existing workflows

## Architecture

```
User Prompt → LLM → Structured Workflow → RPA Executor → Results → LLM Analysis
```

## Integration Patterns

### Pattern 1: Direct Prompt Execution

The simplest integration - user provides a prompt, system executes it:

```python

### Pattern 3: Feedback Loop

Execute workflow, analyze results with LLM, improve:

```python
from ai_rpa_system import WorkflowExecutor, WorkflowManager

def execute_with_llm_feedback(workflow_name: str, llm_analyze_function):
    """
    Execute workflow and get LLM feedback for improvement.
    
    Args:
        workflow_name: Name of workflow to execute
        llm_analyze_function: Function that takes execution result and returns suggestions
    """
    executor = WorkflowExecutor()
    manager = WorkflowManager()
    
    # Execute workflow
    result = executor.execute_workflow_by_name(workflow_name)
    
    # Get LLM-friendly summary
    summary = result.to_llm_summary()
    
    # Send to LLM for analysis
    llm_suggestions = llm_analyze_function(summary)
    
    # LLM can suggest:
    # - Increasing wait times
    # - Adding error handling
    # - Optimizing step order
    # - Adding retry logic
    
    return {
        "result": result,
        "suggestions": llm_suggestions
    }
```

### Pattern 4: Conversational Automation

Multi-turn conversation with LLM to build and refine workflows:

```python
class ConversationalRPA:
    def __init__(self):
        self.executor = WorkflowExecutor()
        self.manager = WorkflowManager()
        self.current_workflow = None
    
    def process_user_message(self, message: str, llm_function):
        """
        Process user message through LLM and execute appropriate action.
        
        Args:
            message: User's message
            llm_function: Function to call LLM with context
        """
        # Build context
        context = {
            "message": message,
            "available_workflows": self.manager.list_workflows(),
            "current_workflow": self.current_workflow.name if self.current_workflow else None
        }
        
        # Get LLM decision
        llm_response = llm_function(context)
        
        # Execute based on LLM decision
        if llm_response["action"] == "execute_workflow":
            result = self.executor.execute_workflow_by_name(
                llm_response["workflow_name"],
                variables=llm_response.get("variables", {})
            )
            return result.to_llm_summary()
        
        elif llm_response["action"] == "create_workflow":
            workflow_json = llm_response["workflow_definition"]
            workflow = Workflow(**json.loads(workflow_json))
            self.manager.save_workflow(workflow)
            return f"Created workflow: {workflow.name}"
        
        elif llm_response["action"] == "modify_workflow":
            # LLM suggests modifications
            pass
        
        return llm_response.get("message", "Action completed")
```

## Example: Complete LLM Integration

```python
import json
from ai_rpa_system import WorkflowExecutor, Workflow

class LLMRPAIntegration:
    """
    Complete integration between LLM and RPA system.
    """
    
    def __init__(self, llm_client):
        """
        Args:
            llm_client: Your LLM client (OpenAI, Anthropic, etc.)
        """
        self.llm = llm_client
        self.executor = WorkflowExecutor()
    
    def execute_natural_language_task(self, user_request: str):
        """
        Execute a task described in natural language.
        """
        # Step 1: Generate workflow using LLM
        prompt = f"""
        Convert this request into an RPA workflow JSON:
        
        Request: {user_request}
        
        Generate a valid JSON workflow with name, description, and steps.
        """
        
        llm_response = self.llm.generate(prompt)
        workflow_json = self._extract_json(llm_response)
        
        # Step 2: Execute the workflow
        workflow = Workflow(**json.loads(workflow_json))
        result = self.executor.execute_workflow(workflow)
        
        # Step 3: Analyze results with LLM
        if not result.success:
            analysis_prompt = f"""
            This RPA workflow failed:
            
            {result.to_llm_summary()}
            
            Suggest how to fix it.
            """
            
            suggestions = self.llm.generate(analysis_prompt)
            return {
                "success": False,
                "result": result,
                "suggestions": suggestions
            }
        
        return {
            "success": True,
            "result": result,
            "message": "Task completed successfully"
        }
    
    def _extract_json(self, llm_response: str) -> str:
        """Extract JSON from LLM response."""
        # Handle markdown code blocks
        if "```json" in llm_response:
            start = llm_response.find("```json") + 7
            end = llm_response.find("```", start)
            return llm_response[start:end].strip()
        elif "```" in llm_response:
            start = llm_response.find("```") + 3
            end = llm_response.find("```", start)
            return llm_response[start:end].strip()
        return llm_response.strip()

## Best Practices

### 1. Structured Output

Always request structured JSON from LLMs:

```python
prompt = """
Generate a workflow in this EXACT JSON format:
{
  "name": "string",
  "description": "string",
  "steps": [...]
}

Do not include any other text, only valid JSON.
"""
```

### 2. Validation

Validate LLM-generated workflows before execution:

```python
from pydantic import ValidationError

def validate_and_execute(llm_json: str):
    try:
        workflow_data = json.loads(llm_json)
        workflow = Workflow(**workflow_data)  # Pydantic validation
        return executor.execute_workflow(workflow)
    except (json.JSONDecodeError, ValidationError) as e:
        return {"error": f"Invalid workflow: {e}"}
```

### 3. Safety Limits

Implement safety checks:

```python
def safe_execute(workflow: Workflow):
    # Limit workflow length
    if len(workflow.steps) > 50:
        return {"error": "Workflow too long (max 50 steps)"}
    
    # Check for dangerous actions
    dangerous_keys = ["delete", "format", "shutdown"]
    for step in workflow.steps:
        if step.text and any(key in step.text.lower() for key in dangerous_keys):
            return {"error": "Potentially dangerous action detected"}
    
    return executor.execute_workflow(workflow)
```

### 4. Feedback Loop

Always provide execution results back to LLM:

```python
def execute_with_feedback(user_request: str, llm_client):
    # Generate workflow
    workflow_json = llm_client.generate(f"Create workflow for: {user_request}")
    
    # Execute
    result = execute_workflow(workflow_json)
    
    # Provide feedback
    feedback = f"""
    Workflow execution result:
    {result.to_llm_summary()}
    
    Based on this result, should we:
    1. Consider it successful
    2. Retry with modifications
    3. Try a different approach
    """
    
    next_action = llm_client.generate(feedback)
    return next_action
```

### Pattern 5: Multi-Agent Collaboration

class MultiAgentRPA:
    """
    Multiple LLM agents collaborate on complex automation.
    """
    
    def __init__(self, planner_llm, executor_llm, reviewer_llm):
        self.planner = planner_llm
        self.executor_llm = executor_llm
        self.reviewer = reviewer_llm
        self.rpa_executor = WorkflowExecutor()
    
    def execute_complex_task(self, task_description: str):
        # Agent 1: Plan the workflow
        plan = self.planner.generate(f"Create a plan for: {task_description}")
        
        # Agent 2: Generate detailed workflow
        workflow_json = self.executor_llm.generate(f"Create workflow from plan: {plan}")
        
        # Agent 3: Review and improve
        review = self.reviewer.generate(f"Review this workflow: {workflow_json}")
        
        # Execute final workflow
        workflow = Workflow(**json.loads(workflow_json))
        return self.rpa_executor.execute_workflow(workflow)
```

### Pattern 6: Learning from Execution

```python
class LearningRPA:
    """
    System that learns from execution history.
    """
    
    def __init__(self, llm_client):
        self.llm = llm_client
        self.executor = WorkflowExecutor()
        self.execution_history = []
    
    def execute_with_learning(self, task: str):
        # Include history in prompt
        history_context = self._format_history()
        
        prompt = f"""
        Previous executions:
        {history_context}
        
        New task: {task}
        
        Based on previous successes and failures, generate an optimized workflow.
        """
        
        workflow_json = self.llm.generate(prompt)
        result = self.executor.execute_workflow(Workflow(**json.loads(workflow_json)))
        
        # Store result
        self.execution_history.append({
            "task": task,
            "result": result,
            "success": result.success
        })
        
        return result
```
